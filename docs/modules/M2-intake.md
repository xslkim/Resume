# M2 素材录入 — 详细需求

> 框架见 [`../REQUIREMENTS.md`](../REQUIREMENTS.md)。
> **模块职责**：获取用户真实素材。**主路径=问答驱动录入**（系统提问、用户作答、抽取回显确认）；辅路径=手动录入/编辑。产出 raw 条目交 [M3](M3-knowledge-base.md) 编译。

---

## 1. 职责与边界
- **做**：问答会话（提问策略+状态机）、答案抽取为结构化候选、回显确认、写入 raw、触发 Ingest；手动录入/编辑/软删除。
- **不做**：wiki 页面的编译与维护（M3）、生成简历（M4）。提问/抽取的 LLM 调用经 [M7](M7-llm-orchestration.md)。
- **冷启动定位**：手动快速录入与问答驱动**并列为首选路径**——用户可先手动录 1~2 段经历**立即出一份简历**（即时报偿），再用问答深挖提质；问答是"增强"而非"前置门槛"。
- **raw 写入**：raw 由应用代码在用户确认后写，LLM 不写（配合 M3-FR-15）。

## 2. 功能需求

| M2-FR-02 | 提问前读当前 wiki（+可选目标/JD），识别缺口、决定下一个问什么，避免重复提问。 | MVP |

| M2-FR-10 | 跨经历缺口回访（基于已有 wiki 主动追问薄弱项）。 | 后续 |
| M2-FR-11 | 按目标画像定制问题集。 | 后续 |

### 2.2 录入路径（导入 / 手动 / 问答，三者并列为冷启动入口）
| 编号 | 需求 | 优先级 |
|---|---|---|
| M2-FR-12 | **导入已有简历 / LinkedIn 导出**（冷启动最高频入口）：上传旧简历（PDF/Word）或 LinkedIn 导出 → **LLM 解析为 raw 候选 → 回显确认（GR-1/GR-8）→ 入库编译**。立即满足"录 1~2 段即出稿"，省去全部问答。 | MVP |
| M2-FR-13 | 手动快速录入/补充经历、成就、技能、证书。 | MVP |
| M2-FR-14 | **basics 采集**：首次/缺口驱动地采集 basics 必选字段（name/email/phone/location/headline）；缺 basics 无法渲染 Header（见 §4 INIT 检查、M3 §3.6）。 | MVP |
| M2-FR-15 | 素材可编辑、软删除（可恢复）。**软删以 `deleted: true` 标记承载，Ingest 读取并跳过（M3-FR-18）**；raw 文件本身仍保留历史（git），不破坏不可变性。 | MVP |
| M2-FR-16 | 素材带录入/更新时间戳。 | MVP |
| M2-FR-17 | 预置标签分类（技能/行业/资深度/软技能），可自定义、可多标签。 | MVP |
| M2-FR-18 | **指标回忆引导**（问答的主动能力，非防守）：用量级选择题/时间线提示帮用户**找回遗忘的真实指标**（"日均交易大概几千还是几万？"），把脑中模糊但真实的数字挖出来——这是问答相对手动录入的核心价值。 | MVP |

## 3. 提问策略
**层次（漏斗）**：L1 骨架（列全经历）→ L2 展开（做了什么）→ L3 成就（结果/影响）→ L4 量化（数字）→ L5 技能（工具/技术）→ L6 缺口（后续，对照目标补）。
**选题优先级（下一个问什么）**：骨架缺失 > 无成就 > 未量化 > 技能未关联 > 缺口；跳过已覆盖与已标"无/跳过"；每轮只问 1 焦点、承接已有答案。
**质量规则**：澄清不诱导（"用户规模什么量级？"✅ / "应该有几百万吧？"❌）；开放式挖叙事、封闭式确认事实；允许"不知道/没有/跳过"且不立即重复问；一问一概念。
**抽取与回显**：区分"用户原话事实"与"系统推断"，推断须确认、歧义须追问不臆测；回显结构化条目供改/删/确认，确认后才写 raw 并 Ingest。

## 4. 会话状态机
| 状态 | 含义 | 主要转移 |
|---|---|---|
| INIT | 加载 wiki 状态+可选目标，**检查 basics 完整性**（缺必选则先引导补 basics），算待办 | → SKELETON |
| SKELETON | 收集经历骨架 | 捕获新经历→CONFIRM；骨架够/用户说没了→DEPTH |
| DEPTH | 按优先级逐层深挖某经历 | 得到答复→CONFIRM；本焦点挖完→重选焦点→DEPTH |
| CONFIRM | 回显候选待确认/编辑 | 确认→INGEST；否决→丢弃回来源；需澄清→回提问 |
| INGEST | 写 raw + 触发 M3 编译 | 回 SKELETON 或 DEPTH |
| GAP（后续） | 目标/JD 驱动缺口追问 | 同 DEPTH 循环 |
| PAUSED | 用户中断，保存状态 | 续答→回最后状态 |
| DONE | 输出小结 + 建议下一步 | 终态 |

核心循环：
```
INIT → 计算待办
loop:
  按优先级选焦点 → 生成 1 问/一簇(ASK)
  用户作答/跳过/没有(ANSWER)
  LLM 抽取候选(EXTRACT) → 回显(CONFIRM)
    确认/编辑 → INGEST(写 raw + 编译) → 更新覆盖集
    否决 → 丢弃或改问
  重算待办；无高优先待办或用户结束 → DONE
  用户中断 → PAUSED(可续)
```
**终止（任一）**：骨架完整且每段经历≥1 成就、其中≥1 量化、技能已关联、无高优先待办；或用户结束；或达单次会话提问预算。
**降级（GR-6）**：CLI/LLM 失败时保存已确认内容、会话可续。

## 5. 提问 prompt 模板（由 M7 固化于 schema.md）
- **(a) 系统角色**：澄清不诱导 / 一次一焦点 / 可跳过 / 产出需确认。
- **(b) 选题+提问**：注入 `{{index_summary}} {{covered_set}} {{skipped_set}} {{target}}` → 输出 focus + question 或 done。
- **(c) 抽取候选**：注入 `{{question}} {{answer}} {{related_pages}}` → 输出 JSON（experiences/bullets/skills/tags，标 fact|inferred、双语、needs_confirm）。
- **(d) 确认后编译**：注入 `{{confirmed_items}}` → 编译进 wiki、标记矛盾、原始问答存 raw。

## 6. 接口与数据
- 输入：用户答复 / 手动表单。
- 输出：raw 条目（→ M3 Ingest）；会话状态（→ M8 `interview_sessions`）。
- 数据契约：抽取 JSON 含 `source="interview:<session_id>"`、`{zh,en}`、`fact|inferred`、`needs_confirm`。
- **会话状态字段**（持久化于 M8 `interview_sessions`）：`state`、`current_focus`（正在深挖的经历/主题）、`target`（可选画像/JD）、`pending_extraction`（待确认候选）、`covered_set`/`skipped_set`（提问 prompt 注入用）、`asked_set`（服务端防重复）、提问预算计数、时间戳。

## 7. 依赖
- M7（提问/抽取/编译的 CLI 调用）、M3（Ingest、读 wiki 状态）、M6（回显确认即 GR-8）、M8（会话状态、raw 文件）。

## 8. 验收标准
- ✅ 能对空白/已有素材提问，自然语言作答后抽取**回显确认**再入库。
- ✅ 不重复问已覆盖/已跳过项；提问不诱导。
- ✅ 会话可中断续答。
- ✅ 手动录入/编辑/软删除可用。
